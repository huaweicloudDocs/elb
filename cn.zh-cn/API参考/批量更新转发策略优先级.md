# 批量更新转发策略优先级<a name="BatchUpdatePoliciesPriority"></a>

## 功能介绍

批量更新转发策略的优先级。

## 接口约束

该接口只用于更新action为REDIRECT\_TO\_LISTENER的独享型实例下的转发策略。

## 调试<a name="atuogenerate_1"></a>

您可以在[API Explorer](https://apiexplorer.developer.huaweicloud.com/apiexplorer/doc?product=ELB&api=BatchUpdatePoliciesPriority)中调试该接口，支持自动认证鉴权。API Explorer可以自动生成SDK代码示例，并提供SDK代码示例调试功能。

<p>当关联的listener开启了enhance_l7policy_enable，且不传该字段， 则新创建的转发策略的优先级的值为：同一监听器下已有转发策略的优先级的最大值+1。 因此，若当前已有转发策略的优先级的最大值是10000，新创建会因超出取值范围10000而失败。 此时可通过传入指定priority，或调整原有policy的优先级来避免错误。 若监听器下没有转发策略，则新建的转发策略的优先级为1。</p>

<p>最小值：<strong>1</strong></p>
